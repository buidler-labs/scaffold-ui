# useMirrorNodeAccount Hook

The `useMirrorNodeAccount` hook is the **single source of truth** for Hedera account data from the Mirror Node. It accepts either an EVM address (`0x...`) or a native account ID (`0.0.XXXXX`) and returns identity, key type, and balance in one cached request. Multiple components using the same input share one fetch via React Query.

## Import

```tsx
import {
  useMirrorNodeAccount,
  type HederaAccount,
  type HederaKeyType,
  type UseMirrorNodeAccountOptions,
} from "@scaffold-hbar-ui/hooks";
```

## Usage

```tsx
const { data, isLoading, isError, refetch } = useMirrorNodeAccount("0.0.2", {
  network: "testnet",
});

// Or with chainId (296 = testnet, 295 = mainnet)
const result = useMirrorNodeAccount(accountIdOrEvm, { chainId: 296 });

// Skip the request when input is undefined
const result = useMirrorNodeAccount(undefined);
// → data: null, isLoading: false
```

## Parameters

| Parameter | Type                          | Default | Description                                                                                     |
| --------- | ----------------------------- | ------- | ----------------------------------------------------------------------------------------------- |
| `input`   | `string \| undefined`         | -       | EVM address (`0x...`) or Hedera account ID (`0.0.12345`). When `undefined`, no request is made. |
| `options` | `UseMirrorNodeAccountOptions` | `{}`    | Optional config (see below).                                                                    |

### Options

| Option    | Type                     | Description                                                                     |
| --------- | ------------------------ | ------------------------------------------------------------------------------- |
| `network` | `"testnet" \| "mainnet"` | Override network (otherwise derived from `chainId`).                            |
| `chainId` | `number`                 | Chain ID to derive network (296 = testnet, 295 = mainnet). Defaults to testnet. |

## Return Values

| Property    | Type                    | Description                                                       |
| ----------- | ----------------------- | ----------------------------------------------------------------- |
| `data`      | `HederaAccount \| null` | Resolved account data, or `null` when not found / not yet loaded. |
| `isLoading` | `boolean`               | `true` while the request is in flight.                            |
| `isError`   | `boolean`               | `true` when the request failed.                                   |
| `refetch`   | `() => void`            | Trigger a new fetch (e.g. after an error).                        |

## HederaAccount Type

| Field        | Type                             | Description                                                       |
| ------------ | -------------------------------- | ----------------------------------------------------------------- |
| `accountId`  | `string`                         | Hedera account ID (e.g. `"0.0.12345"`).                           |
| `evmAddress` | `string \| null`                 | EVM alias if present; `null` for ED25519-only accounts.           |
| `keyType`    | `"ED25519" \| "ECDSA_SECP256K1"` | Account's public key type.                                        |
| `balance`    | `bigint`                         | Balance in **tinybars** (8 decimals).                             |
| `canSignEVM` | `boolean`                        | `true` when the account has an EVM alias (`evmAddress !== null`). |

## Mirror Node

The hook calls the **Hedera mirror node directly**. The mirror host follows the logical network from `options.network` or `chainIdToHederaNetwork(chainId)`:

- **mainnet** → `https://mainnet.mirrornode.hedera.com/api/v1/accounts/{input}`
- **testnet** and **local** → `https://testnet.mirrornode.hedera.com/api/v1/accounts/{input}`

## Caching

The hook uses **React Query** with:

- **Query key:** `["mirrorNode", "account", normalizedInput, network]`
- **staleTime:** 30 seconds
- **gcTime:** 5 minutes

Components that use the same `input` and `network` share one request.

## Example

```tsx
function AccountSummary({ accountIdOrEvm }: { accountIdOrEvm: string }) {
  const { data, isLoading, isError, refetch } = useMirrorNodeAccount(accountIdOrEvm, {
    network: "testnet",
  });

  if (isLoading) return <span>Loading…</span>;
  if (isError || !data) return <button onClick={() => refetch()}>Retry</button>;

  const hbar = Number(data.balance) / 1e8;
  return (
    <div>
      <p>Account: {data.accountId}</p>
      <p>EVM: {data.evmAddress ?? "—"}</p>
      <p>Key: {data.keyType}</p>
      <p>Balance: {hbar.toFixed(4)} HBAR</p>
    </div>
  );
}
```

A full working demo is in the [example app](https://github.com/buidler-labs/scaffold-hbar-ui/tree/main/example).
