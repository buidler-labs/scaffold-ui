# useCreateToken Hook

`useCreateToken` submits a **Hedera Token Service (HTS) fungible token create** via [`useNativeTransaction`](/hooks/useNativeTransaction). It builds a `TokenCreateTransaction` (0 decimals, infinite supply, treasury = your chosen account), validates name/symbol/supply, and returns metadata plus `transactionId`.

The hook does **not** read the connected account by itself. You pass **`getTreasuryAccountId`** so the treasury is always explicit (typically the connected account’s `0.0.x` id).

## Peer dependency

Requires **`@hiero-ledger/sdk`** (same major as your app).

```bash
pnpm add @hiero-ledger/sdk
```

## Import

```tsx
import {
  useCreateToken,
  type CreateTokenParams,
  type CreateTokenResult,
  type UseCreateTokenOptions,
} from "@scaffold-hbar-ui/hooks";
```

## Usage

```tsx
import { useCreateToken } from "@scaffold-hbar-ui/hooks";
import { useHederaSigner } from "./useHederaSigner";

function CreateTokenPanel() {
  const { requireProvider } = useHederaSigner();
  const createToken = useCreateToken({
    getTreasuryAccountId: () => requireProvider().accountId,
  });

  return (
    <button
      type="button"
      disabled={createToken.isPending}
      onClick={() =>
        createToken.mutate({
          name: "My Token",
          symbol: "MTK",
          initialSupply: "1000",
        })
      }
    >
      {createToken.isPending ? "Creating…" : "Create token"}
    </button>
  );
}
```

## Options

| Option                 | Type           | Description                                                                      |
| ---------------------- | -------------- | -------------------------------------------------------------------------------- |
| `getTreasuryAccountId` | `() => string` | Return the Hedera account id (`0.0.n`) to use as treasury; throw if unavailable. |

## Mutation variables (`CreateTokenParams`)

| Property         | Type     | Default | Description                                         |
| ---------------- | -------- | ------- | --------------------------------------------------- |
| `name`           | `string` | —       | Token name (trimmed, max 100 characters).           |
| `symbol`         | `string` | —       | Trimmed, uppercased; must match `^[A-Z0-9]{1,10}$`. |
| `initialSupply?` | `string` | `"0"`   | Non-negative integer as string or number-like.      |

## Token shape (library defaults)

The built transaction uses:

- **Type:** fungible common
- **Decimals:** `0`
- **Supply type:** infinite
- **Treasury:** `AccountId.fromString(getTreasuryAccountId())`

Adjust the SDK in your own code if you need different decimals or supply type.

## Return value

Returns a **TanStack Query `useMutation`** result.

On success, **`data`** (`CreateTokenResult`) is:

| Property            | Type     | Description                          |
| ------------------- | -------- | ------------------------------------ |
| `transactionId`     | `string` | From the wallet.                     |
| `name`              | `string` | Normalized name.                     |
| `symbol`            | `string` | Normalized symbol.                   |
| `initialSupply`     | `number` | Parsed supply.                       |
| `treasuryAccountId` | `string` | Value from `getTreasuryAccountId()`. |

## Validation errors

The mutation throws if:

- Name or symbol missing, name too long, or symbol not 1–10 uppercase alphanumeric.
- `initialSupply` is not a finite non-negative integer.

## Other errors

- **`CapabilityError`** — No native signer ([`useNativeTransaction`](/hooks/useNativeTransaction)).
- **`Error("No transactionId returned from wallet")`** — Empty wallet response.
- Any error thrown from **`getTreasuryAccountId`** (e.g. wallet not connected).
