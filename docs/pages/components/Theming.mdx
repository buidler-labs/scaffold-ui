# Theming

:::info
Before reading this page, we recommend checking out the [Styling](/components/Styling) page to understand the basics of styling Scaffold-UI components.
:::

## Typography (Hedera brand alignment)

Hedera’s brand guidelines specify **Styrene A** as the primary typeface (commercial license required) and **Montserrat** as the replacement font for broader use (presentations, documents, and web when Styrene is not licensed).

**This library**

- Exposes **`--font-sui-family`**. Wrapped components use the `font-sui` utility (`font-family: var(--font-sui-family)`).
- **Default in `@scaffold-ui/components`:** stack starts with **Montserrat** (load it yourself, e.g. [Google Fonts](https://fonts.google.com/specimen/Montserrat)) so open-source consumers stay aligned without bundling Styrene.
- **With a Styrene license:** set the same stack your brand team approves, e.g.  
  `:root { --font-sui-family: "Styrene A Web", "Montserrat", ui-sans-serif, system-ui, sans-serif; }`  
  after loading `@font-face` for Styrene.

**Import order:** load font CSS (or `next/font`) first, then `@scaffold-ui/components/styles.css`, then any overrides to `:root` / `.dark`.

## Dark Theme

All Scaffold-UI component styles use theme variables defined in [packages/components/src/styles.css](https://github.com/buidler-labs/scaffold-ui/blob/main/packages/components/src/styles.css).

The stylesheet includes dark theme styles that automatically apply based on your theme configuration.

```css
@import "tailwindcss";

@custom-variant dark (&:where(.dark, .dark *, [data-theme=dark], [data-theme=dark] *));

...

@variant dark {
  --color-sui-primary: #212638;
  ...
}

```

To enable dark mode, add either a `.dark` class or a `[data-theme=dark]` attribute to a parent element (typically the `<html>` or `<body>` tag). This will automatically apply the dark variant colors to all child components.

## Customizing Theme Colors Globally

You can override the default theme colors by redefining CSS variables in your global stylesheet **after** importing `@scaffold-ui/components/styles.css`.

```css
/* Override Scaffold UI theme colors */
:root {
  --color-sui-primary: #ff4444;
  --color-sui-primary-subtle: #ff9999;
  ...
}

/* Dark mode - explicit selectors matching the custom-variant definition */
.dark,
[data-theme="dark"] {
  --color-sui-primary: #654321;
  --color-sui-primary-subtle: #987654;
  ...
}

```

Purple / blue accents and dark surfaces in Hedera marketing can be approximated with the defaults already tuned in `styles.css`; adjust `--color-sui-primary`, `--color-sui-accent`, and related tokens to match your approved brand palette.

## Token reference (`@scaffold-ui/components`)

Defined in [styles.css](https://github.com/buidler-labs/scaffold-ui/blob/main/packages/components/src/styles.css) (source of truth). Light values live in `@theme`; dark overrides in `@variant dark`.

| Token                               | Role                                                          |
| ----------------------------------- | ------------------------------------------------------------- |
| `--font-sui-family`                 | Font stack for all `DefaultStylesWrapper` / `font-sui` usage  |
| `--color-sui-primary`               | Primary brand / actions                                       |
| `--color-sui-primary-subtle`        | Subtle surfaces, borders                                      |
| `--color-sui-primary-neutral`       | Neutral surface                                               |
| `--color-sui-primary-content`       | Text/icons on primary                                         |
| `--color-sui-accent`                | Accent                                                        |
| `--color-sui-base-content`          | Main body text                                                |
| `--color-sui-success`               | Success                                                       |
| `--color-sui-warning`               | Warning                                                       |
| `--color-sui-error`                 | Error (e.g. toasts); inputs may use primary for invalid state |
| `--color-sui-input-border`          | Input border                                                  |
| `--color-sui-input-background`      | Input background                                              |
| `--color-sui-input-text`            | Input text                                                    |
| `--color-sui-input-placeholder`     | Placeholder                                                   |
| `--color-sui-input-border-error`    | Invalid input border                                          |
| `--color-sui-input-border-disabled` | Disabled input border                                         |
| `--color-sui-skeleton-base`         | Skeleton base                                                 |
| `--color-sui-skeleton-highlight`    | Skeleton highlight                                            |

## `@scaffold-ui/debug-contracts`

That package imports component styles and adds more tokens (e.g. `--color-sui-base-100`, `--color-sui-neutral`, `--color-sui-neutral-content`) for forms and buttons. Override those in your global CSS **after** `@scaffold-ui/debug-contracts/styles.css` if you use the debugger UI.

## Customizing fonts globally

```css
@import url("https://fonts.googleapis.com/css2?family=Montserrat:wght@400;600;700&display=swap");
@import "@scaffold-ui/components/styles.css";

:root {
  --font-sui-family: "Montserrat", ui-sans-serif, system-ui, sans-serif;
}
```

With Styrene (licensed) + Montserrat:

```css
:root {
  --font-sui-family: "Styrene A Web", "Montserrat", ui-sans-serif, system-ui, sans-serif;
}
```

(Define `@font-face` for Styrene per your license; do not rely on unlicensed third-party font hosts in production.)
