# HederaAddress Component

The `HederaAddress` component displays a Hedera address with automatic **bidirectional resolution**. Pass a single `value` — either an EVM address (`0x…`) or a native Hedera account ID (`0.0.n`) — and the component resolves the counterpart via the mirror node. It includes a copy button and a link to the block explorer (HashScan for Hedera chains).

## Behavior

- **With EVM address (`0x…`):** Resolves to Hedera account ID via the host's `/api/hedera?evm=...` (or custom resolver). Shows account ID as primary, optional "EVM: 0x..." as secondary. While resolving, shows "Resolving Hedera Account ID…".
- **With native account ID (`0.0.n`):** Resolves to EVM address via `/api/hedera?accountId=...` (or custom resolver). Shows account ID as primary, resolved EVM as secondary. While resolving, shows "Resolving EVM address…".
- **Explorer link:** Uses Hedera-aware URLs (e.g. HashScan `/account/` for Hedera chains) via `getBlockExplorerAddressLink` from `@scaffold-ui/hooks`. Pass **`blockExplorerAddressLink`** to override the URL used for the primary line's link.
- **Invalid value:** If `value` is non-empty but matches neither a valid EVM address nor a valid `0.0.n` pattern, the component shows an error row.

The host app must provide `GET /api/hedera` (or set the appropriate resolvers from `@scaffold-ui/hooks`) for bidirectional resolution.

## Import

```tsx
import { HederaAddress } from "@scaffold-ui/components";
```

## Props

| Prop                       | Type                                                       | Default   | Description                                                                                 |
| -------------------------- | ---------------------------------------------------------- | --------- | ------------------------------------------------------------------------------------------- |
| `value`                    | `string`                                                   | -         | EVM address (`0x…`) or native Hedera account ID (`0.0.n`). Counterpart is auto-resolved.    |
| `chain`                    | `Chain`                                                    | required  | Blockchain network (e.g. `hederaTestnet`) for resolution and explorer link.                 |
| `format`                   | `"short" \| "long"`                                        | `"short"` | Display format for the address (truncated vs full).                                         |
| `size`                     | `"xs" \| "sm" \| "base" \| "lg" \| "xl" \| "2xl" \| "3xl"` | `"base"`  | Typography and avatar size.                                                                 |
| `disableAddressLink`       | `boolean`                                                  | `false`   | Disable the link to the block explorer.                                                     |
| `avatarComponent`          | `ComponentType<{ address, size, ensImage }>`               | -         | Optional component to render an avatar (e.g. blockie) instead of the default placeholder.   |
| `showEvmAddress`           | `boolean`                                                  | `true`    | When account ID is primary, show "EVM: 0x..." as secondary.                                 |
| `style`                    | `CSSProperties`                                            | -         | Custom styles on the root wrapper (memoize to prevent re-renders).                          |
| `blockExplorerAddressLink` | `string`                                                   | -         | Overrides the computed block explorer URL for the primary display link.                     |
