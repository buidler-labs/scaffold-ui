# HederaAddressInput Component

The `HederaAddressInput` component is an input for Hedera that accepts **two formats**: EVM addresses (`0x` + 40 hex characters) and **native Hedera account IDs** (e.g. `0.0.12345`). The **text in the field stays what the user typed**; the inline suffix shows the other representation. Use optional `onResolvedEvmChange` when you need a checksummed EVM address for contract calls.

Hedera **long-zero** EVM aliases (e.g. `0x000‚Ä¶001c` for `0.0.28`) are accepted with **non-strict** address checks: explorers may show mixed-case hex that does not satisfy EIP-55, which viem would otherwise reject in strict mode.

## Behavior

| Input format             | Resolution                                               | Display                                                                       | `onChange` / suffix                                                            |
| ------------------------ | -------------------------------------------------------- | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| **EVM** (`0x...`)        | Validated; then resolved to Hedera account ID for suffix | Input stays `0x‚Ä¶`; suffix `‚Üí Resolving‚Ä¶` then `‚Üí 0.0.12345`                   | `onChange` = raw string; copy suffix copies account ID                         |
| **Native** (`0.0.12345`) | Resolved to EVM via mirror/API                           | Input stays `0.0.n`; suffix `‚Üí 0xAbCd‚Ä¶1234` (short); copy copies **full** EVM | `onChange` = raw string; `onResolvedEvmChange` gets checksummed EVM when valid |

- **Errors** (blocking): Invalid EVM length/format, bare 40-char hex without `0x`, malformed `0.0.n`, plain digits without dots (e.g. `123`), random text, account not found on Hedera.
- **Warnings** (non-blocking): Native account with no EVM alias; EVM address with no mirror mapping on this network; mirror verification failure.
- **Disabled:** The field is temporarily disabled only while **resolving native account ID ‚Üí EVM** (so the raw `0.0.n` text does not change mid-flight). While **EVM ‚Üí account ID** is resolving, the input stays enabled; the **pending** border and suffix (`Resolving‚Ä¶`) show progress instead.
- **UI**: `BaseInput` ‚Äî default border when idle or resolved (suffix shows conversion and a **copy** control for the resolved value), **pending** uses a neutral `base-content` tint on the border while resolving, **error** uses the input error token. The outer wrapper sets **`aria-busy`** while a lookup is in progress and there is no blocking error (assistive tech can treat the field as busy during resolution).

The host app must provide resolution endpoints (or custom resolvers from `@scaffold-ui/hooks`):

- **EVM ‚Üí account ID:** same-origin `GET /api/hedera?evm=...&network=testnet|mainnet` returning `{ accountId }`.
- **Account ID ‚Üí EVM:** same-origin `GET /api/hedera?accountId=...&network=...` returning `{ evmAddress }`.

## Import

```tsx
import { HederaAddressInput } from "@scaffold-ui/components";
```

## Props

| Prop                  | Type                                      | Default              | Description                                                                                                               |
| --------------------- | ----------------------------------------- | -------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `value`               | `string`                                  | -                    | Current value: EVM address or native account ID string.                                                                   |
| `onChange`            | `(value: string) => void`                 | -                    | Called with the **raw** input text on every change (native ID or `0x` string).                                            |
| `onResolvedEvmChange` | `(address: Address \| undefined) => void` | -                    | Optional: notified when the resolved checksummed EVM address changes (undefined if invalid / cleared).                    |
| `name`                | `string`                                  | -                    | Name attribute of the input element.                                                                                      |
| `placeholder`         | `string`                                  | `"0.0.12345 or 0x‚Ä¶"` | Placeholder text.                                                                                                         |
| `disabled`            | `boolean`                                 | `false`              | When `true`, disables the input. Otherwise, the input is disabled only during **native ‚Üí EVM** resolution (see Behavior). |
| `style`               | `CSSProperties`                           | -                    | Custom CSS styles (memoize to prevent re-renders).                                                                        |
| `chainId`             | `number`                                  | `296` (testnet)      | Chain ID for resolution (296 = Hedera testnet, 295 = mainnet).                                                            |

## Live Examples

### Basic usage

<DocsProvider>
  <HederaAddressInputExample />
</DocsProvider>

```tsx
import React, { useState } from "react";
import type { Address } from "viem";
import { HederaAddressInput } from "@scaffold-ui/components";

function ParentComponent() {
  const [value, setValue] = useState("");
  const [resolvedAddress, setResolvedAddress] = useState<Address | undefined>(undefined);

  return (
    <div className="flex flex-col gap-4">
      <HederaAddressInput
        value={value}
        onChange={setValue}
        onResolvedEvmChange={setResolvedAddress}
        placeholder="0.0.12345 or 0x‚Ä¶"
      />
      {resolvedAddress && <p className="text-sm text-base-content/70">Resolved EVM: {resolvedAddress}</p>}
    </div>
  );
}
```

### With chainId (mainnet)

```tsx
import { HederaAddressInput } from "@scaffold-ui/components";

<HederaAddressInput
  value={value}
  onChange={setValue}
  chainId={295}
  placeholder="0.0.12345 or 0x‚Ä¶"
/>;
```
